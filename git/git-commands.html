Git コマンド図解ガイド

add / commit / push と HEAD の関係性を視覚的に理解する

4つのエリアとデータフロー

Git では、ファイルが4つの場所を行き来します。各コマンドは「どこからどこへ移動させるか」を指定するものです。

📝
Working Directory
作業ディレクトリ
編集中のファイル
📋
Staging Area
ステージング
(Index)
💾
Local Repository
ローカルリポジトリ
(.git)
☁️
Remote Repository
リモート
(GitHub など)
git add ステージに追加 →
git commit ローカルに記録 →
git push リモートへ送信 →
覚え方: 編集 → add(箱に入れる) → commit(金庫に保存) → push(クラウドへ送る)

HEAD とは何か？

HEAD は「いま自分がいるコミット」を指すポインタです。通常はブランチの最新コミットを指します。

main ブランチ:
a1b2c3
初期化
←
d4e5f6
ログイン追加
←
g7h8i9
バグ修正
←
j0k1l2
README更新
→ HEAD が最新コミット j0k1l2 を指している状態

HEAD の表記法

表記意味
HEAD現在のコミット
HEAD~1 / HEAD^1つ前のコミット
HEAD~33つ前のコミット
HEAD@{1}直前に HEAD が指していた場所(reflog)
Detached HEAD: git checkout <hash> で過去のコミットに直接移動すると、HEAD がブランチから外れた状態になります。この状態でコミットしても、ブランチに紐付かないので失われやすいので注意。

基本コマンド一覧

git init
新しい Git リポジトリを作成します。.git フォルダが生成されます。
$ git init
git clone <url>
リモートリポジトリをローカルに複製します。
$ git clone https://github.com/user/repo.git
git status
変更ファイルとステージング状況を確認します。迷ったらまずコレ。
$ git status
git add <file>
ファイルの変更をステージング領域へ追加します。. で全ファイル対象。
$ git add index.html
$ git add .
git commit -m "msg"
ステージング中の変更をローカルリポジトリに記録し、HEAD が新しいコミットを指します。
$ git commit -m "feat: ログイン機能を追加"
git push
ローカルのコミットをリモートリポジトリへ送信します。
$ git push origin main
git pull
リモートの変更を取得し、ローカルにマージします。fetch + merge と等価。
$ git pull origin main
git fetch
リモートの変更を取得しますが、マージはしません。安全に確認したいときに。
$ git fetch origin
git branch
ブランチの一覧表示 / 作成 / 削除を行います。
$ git branch
$ git branch feature/login
$ git branch -d old-branch
git checkout / switch
ブランチを切り替えます。新しい Git では switch 推奨。
$ git switch main
$ git switch -c feature/new
git merge <branch>
指定ブランチの変更を現在のブランチに統合します。
$ git merge feature/login
git log
コミット履歴を確認します。--oneline --graph で見やすく。
$ git log --oneline --graph --all
git diff
変更の差分を表示します。ステージング前の差分を確認する際に使用。
$ git diff
$ git diff --staged
git reset
HEAD を指定コミットに戻します。--softはコミットのみ取消、--hardは作業ディレクトリもリセット。
$ git reset --soft HEAD~1
$ git reset --hard HEAD~1
git revert <hash>
指定コミットを打ち消す新しいコミットを作ります。履歴を壊さない安全な取り消し。
$ git revert a1b2c3
git stash
作業中の変更を一時的に退避します。ブランチを切り替えたいときに便利。
$ git stash
$ git stash pop

HEAD が動くコマンドまとめ

コマンドHEAD の動き作業ディレクトリ
git commit新しいコミットに進む変化なし
git checkout <branch>指定ブランチの先頭へそのブランチの内容に
git reset --soft HEAD~11つ戻す変化なし (変更は staged のまま)
git reset --mixed HEAD~11つ戻す変化なし (staged が解除)
git reset --hard HEAD~11つ戻す強制的に書き換え
git mergeマージコミットを作って進む統合結果に
git rebase付け替え後の位置へ履歴が書き換わる

git init から git push までの流れ

新規プロジェクトを GitHub に公開するまでの一連の流れを、ステップごとに確認します。各ステップで「ファイルがどのエリアに存在するか」を意識しましょう。

1
📁 プロジェクトフォルダを作成 / 移動
Git 管理したいディレクトリへ移動します。
$ mkdir my-project $ cd my-project
2
🎬 git init — リポジトリを初期化
.git フォルダが作成され、そのディレクトリが Git の管理下になります。この時点ではまだコミットは 1 つもありません。
$ git init $ git branch -M main # デフォルトブランチ名を main に
State: 空のローカルリポジトリが誕生
3
📝 ファイルを作成・編集
README やソースコードなどを作業ディレクトリに用意します。この段階ではファイルは「Untracked(未追跡)」状態です。
$ echo "# My Project" > README.md $ git status # → Untracked files: README.md
Area: Working Directory
4
📋 git add — ステージング領域に追加
次のコミットに含めたい変更を選択します。. を指定するとカレント以下すべてを対象にできます。
$ git add README.md # または全ファイルをまとめて $ git add .
Area: Working Directory → Staging Area
5
💾 git commit — ローカルリポジトリに記録
ステージング中の内容をスナップショットとして保存します。新しいコミットが作られ、HEAD がそのコミットを指します。
$ git commit -m "chore: 初回コミット" # [main (root-commit) a1b2c3d] chore: 初回コミット
Area: Staging → Local Repository / HEAD が進む
6
🔗 git remote add — リモートを登録
GitHub などで作成した空のリポジトリ URL を origin という名前で紐付けます。これがプッシュ先になります。
$ git remote add origin https://github.com/user/my-project.git $ git remote -v # 登録内容を確認
Local ↔ Remote の接続を確立
7
🚀 git push — リモートへ送信
ローカルのコミットをリモートに反映します。初回は -u で上流ブランチを設定しておくと、次回以降は git push だけで OK です。
$ git push -u origin main # 次回以降は ↓ だけで push 可能 $ git push
Area: Local Repository → Remote Repository
✓
🎉 完了 — GitHub に公開
ブラウザでリポジトリを開くと、コミットした内容が反映されています。以降は「編集 → add → commit → push」のサイクルを回すだけです。
エリアの移動イメージ: Working Directory ─ add → Staging ─ commit → Local ─ push → Remote
よくあるつまずき: git push で rejected と出るときは、リモート側に先に存在するコミット(README など)との差分が原因です。git pull --rebase origin main で取り込んでから再 push しましょう。

日常の開発フロー (2 回目以降)

# 1. 最新を取得
$ git pull origin main

# 2. 作業ブランチを作成
$ git switch -c feature/login

# 3. ファイルを編集 → 状況を確認
$ git status

# 4. ステージング
$ git add .

# 5. コミット (HEAD が新コミットへ移動)
$ git commit -m "feat: ログイン画面を追加"

# 6. リモートへプッシュ
$ git push origin feature/login

# 7. GitHub で Pull Request を作成してマージ
