VSCode キーボードショートカット集

マウスから手を離さないための、カテゴリ別チートシート

はじめに

VSCode はキーボードだけでほぼ全ての操作が完結するエディタです。 ショートカットを覚えるほどコンテキストスイッチが減り、コードを書くことそのものに集中できます。

このページは、実務で頻度の高いショートカットをカテゴリ別にまとめたチートシートです。 全部覚える必要はなく、「今日から 3 つだけ使ってみる」くらいの距離感が良いです。

OS について: 本ページは Ctrl ベース（Windows / Linux）で記載しています。macOS では原則 Ctrl → Cmd、Alt → Option に読み替えてください。

まず押さえる：記法の読み方

VSCode のショートカットには 2 種類の押し方があります。この違いを知らないと、ドキュメントを読んでも再現できません。

同時押し
Ctrl+Shift+V
Ctrl + Shift + V
3 つのキーを同時に押す。+ で繋がっている。
チョード（順次押し）
Ctrl+K, Ctrl+O
Ctrl + K , Ctrl + O
前半を押して一度離し、続けて後半を押す。, で区切られている。
補足: チョード（和音）はピアノの和音が由来。Ctrl+K を「プレフィックス」として大量のショートカットを 2 段階で割り当てられる仕組みです。

Ctrl+K,Ctrl+S キーボードショートカット設定を開く 競合確認・変更・検索に

覚えきれないショートカットを「調べる・変える」

暗記する必要はありません。VSCode には 自分のキーバインドを一覧・変更できる画面 が組み込まれています。

ショートカット設定を開く

Ctrl+K,Ctrl+S を押すと「キーボード ショートカット」タブが開きます。

検索ボックスで機能名 or キーを入力

例：markdown preview と入力すると関連するコマンドとキーバインドが一覧表示されます。 逆に ctrl+shift+v のようにキーの組み合わせで検索することも可能。

キーバインドを編集

行頭の鉛筆アイコンをクリック → 新しいキーを押す → Enter で確定。 競合がある場合は警告が出ます。

JSON で一括管理したい場合

画面右上の「keybindings.json を開く」アイコンから直接編集できます。 ドットファイル管理したい人向け。

注意: 拡張機能が独自のショートカットを追加することがあり、既定のキーと競合する場合があります。挙動が思った通りでない時は、まず競合チェックから始めましょう。

まず 3 つだけ覚えるなら

いきなり全部は無理なので、費用対効果の高い 3 つを挙げておきます。

優先ショートカット理由
★1 Ctrl+P ファイル名で一瞬でジャンプ。サイドバーを操作する時間が消える。
★2 Ctrl+Shift+P 機能名で検索できるので、ショートカットを覚えていなくても何でもできる。
★3 Ctrl+D 複数カーソルの入口。一度触ると置換より手軽で戻れなくなる。
