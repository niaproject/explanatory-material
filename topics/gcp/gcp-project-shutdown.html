Google Cloud プロジェクトのシャットダウン

30 日猶予つき「削除予定」状態への移行手順と、停止されるもの・されないもの

「シャットダウン」とは何か

Google Cloud の「シャットダウン (Shut down)」は、プロジェクトを即削除ではなく「削除予定」状態に移す操作です。 30 日間の猶予があり、その間は復元できますが、期間を過ぎると物理削除され、同じプロジェクト ID は二度と使えません。

削除までのタイムライン

Day 0
シャットダウン実行
配下の全リソース（VM / GKE / Cloud Run / BigQuery / Storage 等）への API アクセスが即時遮断。課金も新規発生分は停止。
Day 1 〜 Day 29
削除予定（復元可）
リソースは停止状態で保持される。gcloud projects undelete で復元可能。
Day 30
物理削除
プロジェクトと配下リソースが完全消去。プロジェクト ID は再利用不可。

何が止まる？ / 何が残る？

⛔ 停止されるもの
  • コンピュート系: Compute Engine / GKE / Cloud Run / Cloud Functions がすべて停止
  • ネットワーク: VPC、LB、外部 IP、Cloud DNS レコードが無効化
  • データアクセス: Cloud Storage / BigQuery / Cloud SQL への読み書き不可（データ自体は 30 日保持）
  • API アクセス: 配下の全エンドポイントが 403
  • OAuth クライアント: 発行したクライアント ID が無効化 → 紐づくアプリのログインが失敗
  • サービスアカウント: 鍵も無効化 → CI / バッチが一斉に落ちる
  • IAM バインディング: プロジェクト単位の権限が無効化
  • 課金: 新規のリソース課金は停止
✅ 残るもの
  • 組織 / フォルダ階層: 親階層は影響なし
  • 請求先アカウント: 他プロジェクトで引き続き使える
  • 削除予定中のデータ: 30 日以内なら復元可能
  • 既発生の請求: シャットダウン前の使用分は翌月請求書に計上される
  • 他プロジェクトへの参照: 他プロジェクトに作ったリソースはそのまま

シャットダウンする 3 つの方法

1Cloud Console (GUI)
ブラウザで操作。一番安全で、確認ダイアログもある。初回や不安なときはこれ。
2gcloud CLI
スクリプトで一気にやりたい時。Console と同じ 30 日猶予の動き。
gcloud projects delete PROJECT_ID
3gws CLI 経由
Workspace CLI から Cloud Resource Manager API を叩く方法。
gws cloudresourcemanager \
  projects delete \
  --params '{"projectId":"PROJECT_ID"}'

Cloud Console から削除する手順

Cloud Console を開いてプロジェクトを選択

console.cloud.google.com を開き、画面上部のプロジェクト選択プルダウンから削除したいプロジェクトを選びます。 プロジェクト名と ID を目視確認してください（別プロジェクトの誤削除防止）。

「IAM と管理」→「設定」を開く

左上の ≡（ハンバーガー） からナビゲーションを開き、以下の階層を辿ります。

≡ › IAM と管理 › 設定

「IAM と管理」のサブメニューの一番下の方に「設定」があります。 IAM / PAM / サービス アカウント / ラベル / タグ などと並んでいるので、見落としがちです。

直接 URL でも OK:
https://console.cloud.google.com/iam-admin/settings?project=PROJECT_ID
UI が変わった・メニューが見つからないときはこの URL が確実です。

「シャットダウン」ボタンを押す

設定ページの上部にプロジェクト名 / プロジェクト ID / プロジェクト番号が表示されます。 その近くにある「シャットダウン」（英語 UI では SHUT DOWN）ボタンをクリック。

確認ダイアログにプロジェクト ID を入力

「削除するには以下のプロジェクト ID を入力してください」と表示されるので、プロジェクト ID をそのまま入力します。 プロジェクト名ではなく ID である点に注意。コピペ推奨です。

最終の「シャットダウン」をクリック

「プロジェクトは削除が予定されました」と表示されれば完了です。以降 30 日間は削除予定状態です。

「設定」が見つからないときの代替ルート

「IAM と管理」メニューを開いても「設定」項目が見つからない場合、次の順で試してください。

ルート A: 直接 URL（最速）

https://console.cloud.google.com/iam-admin/settings?project=PROJECT_ID

ルート B: 画面上部の検索バー

Console 上部の検索バーに以下のいずれかを入力し、候補から「IAM と管理 → 設定」を選ぶ。

  • シャットダウン
  • IAM 設定
  • プロジェクトの設定
  • 英語 UI: Shut down / Project settings

ルート C: Resource Manager の一覧から削除

https://console.cloud.google.com/cloud-resource-manager

プロジェクト一覧から対象行の ⋮（三点メニュー） → 「削除」。一覧から直接削除できるルートです。

ルート D: CLI で済ませる

gcloud projects delete PROJECT_ID

UI が不調なとき、最も確実に削除できるのはこちら。

削除できない / ボタンが押せないとき

症状原因対処
ボタンがグレーアウト 自分に roles/owner が無い オーナーに依頼 or 権限付与
「組織のポリシーで禁止」 組織管理者が constraints/resourcemanager.allowDelete で禁止 組織管理者に解除を依頼
「請求先アカウントにリンクされている」警告 警告のみ（ブロックではない） そのまま進めて OK。気になるなら事前に請求リンク解除
「リソースを含むため削除できない」 CMEK キーなど一部リソースが残っている 該当リソースを個別削除してから再試行
メニューに「設定」が出ない 権限不足 / 画面崩れ 上記「ルート A 〜 D」で回避

復元したいとき（30 日以内）

削除予定中なら、同じ設定ページに「復元」ボタンが出ます。CLI なら 1 行:

gcloud projects undelete PROJECT_ID
30 日を過ぎたら: 復元は不可能。しかも同じプロジェクト ID は永久に使えなくなります。 再構築する場合は新しい ID を付ける必要があります。

シャットダウン前チェックリスト

押す前に一度目を通しておくと事故を防げます。

  • プロジェクト ID を控えた（復元や記録のため）
  • 対象プロジェクト配下で発行した OAuth クライアント ID が、他のアプリ・端末で使われていないか確認
  • サービスアカウント鍵 が CI / バッチ / 他プロジェクトに埋め込まれていないか確認
  • BigQuery / Cloud Storage 上に残したいデータがあればエクスポート
  • Cloud DNS / LB / 外部 IP の外部ユーザーへの影響を洗い出し
  • 請求先アカウントで発生済み利用分の翌月請求額を確認
  • 他のプロジェクトやフォルダからこのプロジェクト ID を参照していないか確認

まとめ

  • 「シャットダウン」は30 日猶予つきの削除予定化。即削除ではない。
  • 実行経路は Cloud Console / gcloud projects delete / gws の 3 つ。
  • Console からの場所は「IAM と管理 → 設定」。見つからなければ直接 URL /iam-admin/settings?project=… が最速。
  • 実行直後に OAuth クライアント / サービスアカウント / API アクセスが一斉に止まることを忘れずに。
  • 30 日を過ぎるとプロジェクト ID が永久欠番になる点が一番の落とし穴。
